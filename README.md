# QQQ Buy-Signal Monitor (with Options Wall Confirmation)

一个基于五大维度市场信号（现货+情绪）并引入期权墙（Options Wall）硬否决逻辑的 QQQ 买点监控系统。

## 🌟 核心特性 (v2.0 最新架构)

- **Tier-0 (宏观防御层)**: 实时拉取 FRED 信用利差数据。若发生流动性危机 (利差飙升)，直接拉响红色警报，全系统无条件切断所有买点。
- **Tier-1 (基础情绪层)**: 综合 52周回撤、MA200 偏离度、VIX 恐慌指数、CNN Fear & Greed 指数、市场广度（涨跌比）。
- **Tier-1.5 (背离与估值红利)**: 
  - 动态跟踪 QQQ 的 Forward PE，跌入便宜区间则自动加分，泡沫区间则减分扣点。
  - 60 天滚动底背离引擎，精准捕捉 Price-Breadth, Price-VIX, Price-RSI 的恐慌分歧信号 (🔥 额外红利分)。
- **Tier-2 (期权确认)**: 实时计算 **Put Wall (支撑墙)**、**Call Wall (压力墙)** 与 **Gamma Flip (多空分界点)**。
- **硬否决逻辑 (Standard Rule)**: 即使前序得分再高，若价格跌破 Put Wall 或触发 Tier-0 宏观熔断，系统将强制否决 `TRIGGERED` 信号，转为 `WATCH` 或 `NO_SIGNAL`，有效规避左侧接飞刀。
- **现代化技术栈**: Python 3.12, 异步与无 API Key 数据采集, SQLite 历史与宏观状态持久化, Docker 容器化部署。

## 📊 信号分级

| 状态 | 说明 | 行动建议 |
|:---:|---|---|
| **TRIGGERED** | 现货极端超卖 + 情绪极度恐慌 + 期权墙支撑稳固 | 极高胜率买点，适合底仓入场 |
| **WATCH** | 部分信号达标，或处于跌破支撑后的观察期 | 关注市场企稳，等待期权墙收复 |
| **NO_SIGNAL** | 市场处于常规波动或上涨趋势中 | 继续持有，无需操作 |

## 🚀 快速开始 (Docker)

项目完全兼容 Docker，无需本地配置 Python 环境。

### 1. 运行实时监控
每天收盘后运行，获取当前买点建议：
```bash
docker-compose run --rm app python -m src.main
```

### 2. 运行历史长线回测 (1999-2025)
验证深度底背离与极端情绪的 "钻石坑" 捕捉能力：
```bash
docker-compose run --rm app python src/backtest.py
```

### 3. 运行测试套件
```bash
docker-compose up --build test
```

## 🛠️ 技术细节

### 数据来源
- **价格/均线**: Yahoo Finance (`yfinance`)
- **情绪指数**: CNN Fear & Greed (内部 API)
- **期权链数据**: yfinance (实时数据)
- **市场广度**: NYSE Net Advances (`^ADD`)

### 核心算法
- **Tier-0 熔断**: 信用利差 > 500 bps 时否决买入。
- **Tier-1.5 底背离捕捉**: 本周价格创新低，但对应深层基本盘（VIX、涨跌参与度、RSI动能）没有恶化时，确认“多头抵抗”，叠加额外得分。
- **Gamma Flip**: 通过期权链各行权价的 Gamma 分布进行零点插值计算，定位市场波动性由收缩转为放大的临界点。
- **Support Veto**: 基于做市商 Delta 对冲压力。当价格低于 Put Wall 时，做市商不得不随着下跌卖出更多期货以对冲，形成螺旋下行压力，此时买入风险极大。

## 📈 后台回测表现 (v2.0 阶段)

针对 1999 年至 2025 年长达 6696 个交易日的超长周期回测结果显示：
- **触发极低频率**：系统仅在 **6.0%** (约 400 天) 的极端恐慌、超低估值、且有底背离保护的日子发出 `TRIGGERED`，成功锁定了 2000 年互联网泡沫末期、2008 年次贷危机、2020 年疫情底、2022 年美联储加息底等公认的“钻石坑”。
- **强大的自我过滤**：多达 450 天虽然技术指标严重超卖（通常人们会进行左侧抄底），但被强大的 **期权墙破位否决** 或 **宏观熔断否决** 给硬性拦截了，完美规避了下降趋势中的“半山腰”陷阱。
