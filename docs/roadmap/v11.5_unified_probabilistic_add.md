## 1. 架构总览 (Architectural Overview)

本系统采用 `Pipe-and-Filter` 与 `Hierarchical Neural/Bayesian Inference` 混合架构。彻底解除了宏观状态阈值的硬编码，数据以连续向量形式流经 4 层结构，最终生成具备极强抗污能力的 Beta 指令池。

```mermaid

        Z --> KDE[Gaussian KDE 模型库]

        Diff -- ">= 边界" --> Action(生成调仓与 T+1 通信锁)
    end
```

## 2. 核心模块代码映射与组件设计

### 2.1 多维跨时域数据生成层
* **组件**: `src.engine.v11.probability_seeder`
* **指责**: 强制在进入降维或推演前完成极端的时区（`tz_convert(None)`）与时间切口对齐。确保每日的宏观因子不会发生 Look-Ahead Bias。不同类的因子将强制应用经历史检验的固化滑动窗口。

### 2.2 多核高斯朴素贝叶斯模型 (Multivariate GNB / Kernel Density)
* **组件**: `src.engine.v11.probabilistic_pipeline.BayesianEngine`
* **指责**: 包含过去 25 年数据的预训练 KDE 分布参数（或 GNB 的均值与方差），接收实时传递的标准化向量 $X_t = [f_1, f_2, ..., f_n]^T$。
* **计算**: 输出各个基础宏观组合状态下的边界概率分布 `test_probs`。

### 2.3 信息熵防护器 (Entropy Controller Algorithm)
* **组件**: `src.engine.v11.entropy_controller`
* **数学表达**:
  $$ H_t = -\sum_{i} P_i \log_2(P_i) $$
  $$ Normalized\_H_t = \frac{H_t}{\log_2(5)} $$
  当探测到 $Normalized\_H_t > 0.75$ 时，系统定义当前状态发生特征坍缩（Feature Collapse），应用向 1.0 的安全基线归零算法（Base Beta Regression）。

### 2.4 Hysteresis 滞后阻尼与 Beta 输出
* **组件**: `src.engine.v11.hysteresis_beta_mapper.py`
* **指责**:
  哪怕底层分类体系已经通过贝叶斯网络给出明确建议，如果与**前一交易日 Target Beta 之间的绝对差值不足，它将在阻尼器中被立刻吸收并抛弃**，以最大化限制交易磨损。同时必须耦合 T+1 资金锁：当发生资产层面的买卖时，后续对立方向信号冷却 24 小时才被允许发放。

## 3. 并行回测评测协议

并行评测引擎 (`scripts.parallel_crisis_backtest`) 需要接管这套概率化结构进行大范围校验。核心调用的池化函数必须是高并发安全的线程分配，确保存储历史预测（Brier 分数监控）的 `sqlite` 与 `log` 文件没有死锁。
