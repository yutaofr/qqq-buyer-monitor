# QQQ Buy-Signal Monitor v3.0 - 宏观基本面全天候过滤引擎 (PRD)

## 1. 业务背景与愿景

**系统进化路径**：
*   **v1.0 (MVP) - 恐慌猎手**：基于现货量价、情绪极限（VIX/FG）与微观期权结构（Put Wall），精准捕捉单纯的流动性踩踏底部。
*   **v2.0 (规划中) - 背离专家**：引入时间序列的技术极值比较（Price-RSI / Price-Breadth 底背离），解决指标绝对估值低位钝化的问题。
*   **v3.0 (本蓝图) - 全天候量化底仓防御系统**：纯粹的技术面与筹码面在面临真正的**“企业盈利坍塌（Earnings Recession）”**或**“长期无风险利率飙升体系重估”**时会失效。作为终极对冲基金级别的宏观全视角模型，v3.0 的目标是：摒弃容易失真的传统基本面指标（如 P/E），直接切入反映资金成本、印钞能力与华尔街远期定价的核心重磅宏观锚点。

## 2. 核心大局观指标设计 (Fundamental & Macro Features)

### 2.1 股权风险溢价 ERP (Equity Risk Premium)
**系统逻辑：决定整体仓位宽松与收紧的“时代环境开关 (Regime Switch)”**。

*   **指标释义**：`ERP = QQQ 盈利收益率 (1 / Forward PE) - 10年期美国国债收益率 (US10Y)`。评估承担股市波动的玩家，到底比拿着无风险国债赚取了多少额外补偿。
*   **适用场景**：
    *   **高压警报**：当 ERP 极低（如小于 1% 甚至为负），股市性价比极低。任何轻微的盈利不及预期都会引发踩踏（如 2023 年底的杀估值）。
    *   **历史大底**：当 ERP 极高（如大于 5%），说明股市极度恐慌且极其便宜，大量机构长线资金即将进场配置。
*   **引擎交互 (Regime Shift)**：
    *   当 `ERP < 1.0%`：开启 **防御模式 (Defense Regime)**。系统将 Tier-1 的 `TRIGGERED` 阈值从 70 分上调至极其苛刻的 85 分，或者直接一票否决所有加分项，拒绝入场吃套。
    *   当 `ERP > 5.0%`：开启 **激进模式 (Aggressive Regime)**。系统触发隐藏底层长线做多信号，并在 Discord 输出【百年一遇系统性低估】标签。

### 2.2 自由现金流收益率 (FCF Yield)
**系统逻辑：取代传统 PE 作为科技股最真实的“印钞机折扣率”底座**。

*   **指标释义**：`FCF Yield = QQQ 成份股权重加权自由现金流总额 / QQQ 总市值`。
*   **适用场景**：对于像 Apple, Microsoft, Google 这样的科技巨头，账上庞大的自由现金流决定了他们抗周期的能力（回购与高分红）。相比容易被财务技巧粉饰的净利润（Net Income / PE），自由现金流收益率更能反映“真金白银的打折力度”。
*   **引擎交互 (Valuation Baseline)**：
    *   当 `FCF Yield > 4.5%` (历史黄金击球区)：为 Tier-1 的长期定投得分**无条件增加 15 分奖励（FCF Bonus）**。这意味着即使市场仅仅是横盘微红，系统也会发出 `WATCH` 提示建仓，因为资产实在太便宜了。

### 2.3 盈利预期上修广度 (Earnings Revisions Breadth)
**系统逻辑：过滤“杀业绩假摔”，左侧识别盈利底（Fundamental Turning Point）**。

*   **指标释义**：在 QQQ 成分股中，过去 30 天内被华尔街投行**上调 EPS 预期**的公司数量，占总被调整数量的百分比。
*   **适用场景**：股票跌穿支撑位，究竟是因为恐慌，还是因为真的基本面烂了？如果大盘在阴跌，但你发现分析师却在悄悄集体上调科技股下季度的业绩预测，这就是传说中的“业绩底先于市场底”。
*   **引擎交互 (Fundamental Divergence)**：
    *   当 QQQ 现货价格创新低，但 `Earnings Revisions Breadth > 50%` 且处于上升通道时，触发 **基本面底背离 (Fundamental Divergence)**。
    *   此时如果同时满足 Tier-2 期权墙不跌破，系统直接在 Discord 中发出**最高级别的重仓建仓信号（Strong Buy）**。

## 3. 技术挑战与架构突破

### 3.1 极低频与实时高频的数据揉弦 (Time-Series Alignment)
*   **挑战**：Tier 1 与 Tier 2（现货价、期权链）是**秒级/日级别**更新的快速变量。而 ERP、FCF Yield 和 Earnings Revision 是**周级甚至月级别（随财报季波动）**的缓慢变量。
*   **解决方案**：
    需要引入专门的**低频宏观状态机 (Macro State Machine)**。
    1.  爬虫模块每周末运行一次，抓取 FactSet / Bloomberg（或高质量平替如 YCharts, AlphaVantage 宽基数据集）的宏观宽基基本面数据，落入单独的 `macro_states` 数据库表。
    2.  每日的高频引擎读取最近一次有效的宏观状态字典，作为环境变量（Context）注入到当天的计算引擎中。

### 3.2 高纯度数据源依赖
*   由于免费的 `yfinance` 无法提供深度的机构级前瞻预测数据（Forward EPS, Analyst Revisions）。
*   v3.0 必须考虑商业化数据源的整合，或者通过 NLP (自然语言处理) 定期抓取华尔街宏观研报/彭博头条，进行自动化的“上调广度”打分。

## 4. 演进路线图归纳

在完成此规划后，QQQ Monitor 的最终形态将是一台**拥有极客审美和华尔街视角的六维立体扫描仪**：

1.  **【微观结构】期权做市商护城墙 (Tier-2 Gamma / Options Wall)**：防闪崩、防流动性螺旋。
2.  **【市场情绪】极端恐慌指标 ( Tier-1 VIX / F&G / Breadth)**：寻找散户带血的筹码。
3.  **【技术动能】波段底背离 (v2.0 Price-Divergence)**：动能衰竭与多头反击确认。
4.  **【宏观之锚】股权风险溢价 (v3.0 ERP)**：决定要不要承担风险的大环境开关。
5.  **【基本底座】自由现金流回报 (v3.0 FCF Yield)**：决定资产当前的绝对折扣力度。
6.  **【盈利预期】分析师上修广度 (v3.0 Earnings Revision)**：拆穿杀业绩假摔的谎言。
