# ADD: v8.1 存量资产 Beta 回测可视化实现方案

> 版本: v8.1
> 状态: Final
> 日期: 2026-03-26
> 对应 SRD: `docs/v8.1_stock_beta_backtest_srd.md`

## 1. 实现目标

新增一条独立的 beta 回测可视化链路，复用现有 backtest 结果生成一张双面板图：

- 上面：QQQ 收盘价和系统建议 beta 的同屏对比。
- 下面：beta 建议路径、beta band、关键 regime 标记。

这条链路只负责展示，不改变任何决策输出。

## 2. 设计原则

- 报告逻辑与决策逻辑分离。
- 图表生成可重复、可测试、可从脚本单独再生。
- 输入容错清晰，列名不匹配时直接失败，不做静默猜测。
- 风格与现有增量资金 DCA 图保持一致。

## 3. 模块改动

### 3.1 新增图表模块

#### [CREATE] `src/output/backtest_plots.py`

提供可复用函数：

```python
def build_beta_backtest_figure(daily_ts: pd.DataFrame, summary: BacktestMethodologySummary | None = None):
    ...

def save_beta_backtest_figure(
    daily_ts: pd.DataFrame,
    summary: BacktestMethodologySummary | None,
    output_paths: Sequence[str | Path],
) -> list[Path]:
    ...
```

职责：

- 规范时间轴。
- 选择 `target_beta` / `signal_target_beta` 列。
- 统一黑底风格和图例。
- 输出到多个路径。

### 3.2 新增脚本

#### [CREATE] `scripts/plot_beta_backtest_performance.py`

职责：

- 读取 `data/qqq_history_cache.csv` 和 `data/macro_historical_dump.csv`。
- 运行 `Backtester.simulate_portfolio(..., enable_dynamic_search=True)`。
- 调用 `save_beta_backtest_figure()`。

### 3.3 修改回测入口

#### [MODIFY] `src/backtest.py`

在 `run_backtest()` 的 summary 输出后增加图表保存步骤。

只在 `summary.daily_timeseries` 存在且包含目标列时执行。

### 3.4 文档更新

#### [MODIFY] `docs/backtest_report.md`

增加 beta 回测图的引用，说明它展示的是存量资产 beta 建议，而不是增量部署节奏。

## 4. 测试策略

### 4.1 单元测试

新增图表模块测试，覆盖：

- 当输入包含 `target_beta` 时可以成功生成图。
- 当只有 `signal_target_beta` 时也可以生成图。
- 当缺少 beta 列时会抛出明确错误。
- 图中包含两个子图和关键标签。

### 4.2 集成测试

更新回测入口测试，确保：

- `run_backtest()` 仍然先打印宏观覆盖，再打印回测摘要。
- 新增图表步骤不会破坏现有 monkeypatch 测试。
