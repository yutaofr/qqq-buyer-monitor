## 3. 黑盒测试场景

### QB-V9-001 中性环境

- 输入：tight spread, no stress
- 期望：
  - `risk_state` 为 `RISK_NEUTRAL` 或 `RISK_ON`
  - `target_beta >= 0.9`
  - `deployment_state` 非 `DEPLOY_PAUSE`

### QB-V9-002 紧缩但非危机

- 输入：`RICH_TIGHTENING`
- 期望：
  - `risk_state == RISK_REDUCED`
  - `target_beta <= 0.8`
  - `QLD` 可以有限参与，但不得超过 ceiling

### QB-V9-003 压力防守

- 输入：dual stress / `TRANSITION_STRESS`
- 期望：
  - `risk_state == RISK_DEFENSE`
  - `target_beta <= 0.7`
  - `QLD` 若参与，只能更低限度参与

### QB-V9-004 危机退出

- 输入：`CRISIS`
- 期望：
  - `risk_state == RISK_EXIT`
  - `target_beta == 0.5`
  - `QLD == 0`
  - 非授权时 `deployment_state == DEPLOY_PAUSE`

### QB-V9-005 危机血筹 override

- 输入：满足已认证 blood-chip 条件
- 期望：
  - `deployment_state == DEPLOY_FAST`
  - `target_beta` 不抬升
  - `QLD` 不被授予
