# ADD: v10.0 周期因子决策层

> 版本: v10.0
> 状态: Approved for implementation
> 日期: 2026-03-29
> 对应 SRD: `docs/v10.0_cycle_factor_srd.md`

## 1. 架构决定

v10.0 采用兼容式演进：

1. 保留 `tier0_regime`，用于兼容 v9 历史记录、deployment ceiling 和已有回测列。
2. 新增 `cycle_regime`，作为 v10 的真实上游制度层。
3. `RiskDecision` 不再只由 `tier0_regime + Class A stress` 决定，而是接收 `CycleDecision` 的 ceiling。

## 2. 新增模块

### 2.1 `src/models/cycle.py`

新增：

- `CycleRegime`
- `CycleDecision`

### 2.2 `src/engine/cycle_factor.py`

新增 `decide_cycle_state(snapshot)`。

职责：

- 从统一的 `FeatureSnapshot` 读取估值、信用、广度和趋势
- 输出周期状态与 `QLD` 合法域
- 对缺失数据 fail-closed

## 3. `FeatureSnapshot` 扩展

新增写入字段：

- `erp`
- `price_vs_ma200`

保留并复用：

- `breadth`
- `credit_spread`
- `credit_acceleration`
- `rolling_drawdown`
- `liquidity_roc`
- `funding_stress`

## 4. 风控合并规则

`RiskDecision` 的最终输出遵循：

- `final_exposure_ceiling = min(base_risk_ceiling, cycle_ceiling)`
- `final_qld_share_ceiling = min(base_risk_qld_ceiling, cycle_qld_ceiling)`

同时允许周期层改变风险状态语义：

- `BUST` -> 至少 `RISK_EXIT`
- `LATE_CYCLE` -> 至少 `RISK_REDUCED`
- `CAPITULATION` -> 在没有 drawdown / triple-stress 硬退出时允许 `RISK_ON`

## 5. 候选池要求

v10.0 要求 registry 至少覆盖：

- `RISK_ON` 的高 beta、有限 `QLD` 候选
- `RISK_NEUTRAL` 的 pure-QQQ 候选
- `RISK_NEUTRAL` 的 recovery limited-`QLD` 候选
- `RISK_REDUCED` 的 pure-QQQ 候选
- `RISK_EXIT` 的 `QLD=0` floor 候选

## 6. 叙事输出

runtime trace 升级为：

`Tier-0 -> Cycle -> Risk -> Candidate -> Advisory -> Deployment -> Reference`

要求：

- CLI explanation 标注 v10.0
- Discord embed 显示 `cycle_regime`
- Web snapshot `meta.version` 升级到 `v10.0`
- `index.html` 静态说明文字改为 v10 决策链

## 7. 回测落地

## 8. 风险与缓解

- 风险：历史宏观数据无原生 breadth / MA200
- 缓解：使用与 live 一致的价格侧代理重建，而不引入新的离线数据工程

- 风险：v9 registry 候选不足以表达 v10 周期层
- 缓解：补充 pure-QQQ 和 recovery / capitulation 候选

- 风险：输出面升级造成 narrative 断裂
- 缓解：统一从 `build_runtime_logic_trace()` 生成 CLI / Web / Discord 文本
