# SRD: 战略资产配置与 Beta 风险监控系统 (v6.3)

## 1. 概述
本系统在 v6.2 防御架构的基础上，新增“战略资产配置 (Target Asset Allocation)”维度。通过对现状的审计与对目标的建模，实现“现状 vs 理想”的差分管理。

## 2. 核心设计原则
- **语义分层隔离**：命名上严格区分现状 (`CurrentPortfolioState`) 与目标 (`TargetAllocationState`)。
- **输入鲁棒性**：遵循“防御性输入协议”，对非法、全零或冲突的环境变量进行确定性降级处理。
- **多资产回测闭环**：回测引擎必须支持基于 [Cash, QQQ, QLD] 三资产的单位制仿真。

## 3. 目标配置模型矩阵 (Model Portfolio Matrix)

系统基于 `AllocationState` 映射以下 **TargetAllocationState** 模型：

## 4. 技术协议规范

### 4.1 输入归一化与安全性协议 (Input Safety)
- **读取流**：解析 `CASH_LEVEL`, `QLD_LEVEL`, `QQQ_LEVEL` (默认 0.0)。
- **合法性截断**：所有负值输入强制修正为 0.0。
- **分母安全检查**：计算 $Sum = C_{raw} + Q_{raw} + L_{raw}$。
- **降级路径 (Safe Fallback)**：
    - 若 $Sum \le 0$ 或包含 `NaN/Inf`：强制设定 Reality = [1.0, 0.0, 0.0] (100% 现金)，并触发日志告警。
    - 否则：执行 $Val_i / Sum$ 进行归一化。

### 4.2 QLD 价格合成协议
- **公式**：$P_{qld, t} = P_{qld, t-1} \times (1 + 2 \times R_{qqq, t} - 0.0000377)$。

## 5. 验收标准 (Acceptance Criteria)

- **AC-1**: 验证在全零输入下，系统不报错且输出 `Reality Cash: 100%`。
- **AC-2**: 验证在 L1-L3 防御状态下，Target 模型中的 QLD 权重强制为 0。
- **AC-3 (Sim Integrity)**: 验证回测中每一时刻的 $NAV = \sum(Units \times Price) + Cash$，浮点误差应 $< 10^{-4}$。
- **AC-4 (Beta Fidelity)**: 历史回测中，有效区间的“实现贝塔”与“目标贝塔”的绝对偏差均值 $\le 0.05$。
- **AC-5 (MDD Improvement)**: 在 2008.09 - 2009.03 期间，TAA 建议组合的最大回撤相比基准 QQQ 改善 $\ge 20\%$。
