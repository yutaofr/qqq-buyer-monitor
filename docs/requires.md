QQQ 买点信号加入期权墙确认层

采用 4 个信号判断 QQQ 是否出现买点:

1. 距 52 周高点回撤幅度
2. 距 200 日均线偏离幅度
3. VIX 是否抬升
4. Fear & Greed 是否进入恐慌区
5. 市场参与度

这套规则能较好描述 "价格是否便宜" 和 "市场是否恐慌", 但缺少一个关键维度: 期权市场结构是否支持当前位置形成有效支撑。

在实际交易中, QQQ 是否值得买, 不只取决于现货估值和情绪, 还取决于当前价格相对主要 `put wall`、`call wall` 和 `gamma flip` 所处的位置。缺少这一层时, 系统容易出现两类误判:

1. 现货和情绪信号都偏多, 但价格已经跌破关键 `put wall`, 下方支撑失效, 此时贸然给出买点会提高抄底失败率。
2. 市场进入恐慌, 但价格正好站在强 `put wall` 上方, 同时上方 `call wall` 仍留有空间, 这类位置往往比单纯看 VIX 更接近高性价比买点。

因此, 需要把期权墙引入 QQQ 买点模型, 作为确认层和否决层, 而不是简单增加一个孤立指标。

## 2. 问题定义

现有模型需要回答两个问题:

1. QQQ 便不便宜
2. 市场慌不慌

然后回答第三个问题:

3. 期权市场是否允许当前位置形成可交易的反弹支点

产品体现在三方面:

1. 缺少对下方支撑是否真实存在的判断
2. 缺少对上方阻力是否过近的判断
3. 缺少对 "可以买"、"该观察"、"先别接" 三种状态的区分

## 3. 产品目标

### 3.1 目标

1. 实现 QQQ 买点框架并加入期权墙确认层, 降低抄底型误报。
2. 将二元判断升级为三态判断: `触发`、`观察`、`未触发`。
3. 保持结果可解释, 输出必须明确说明 `put wall`、`call wall`、`gamma flip` 对结论的影响。

### 3.2 非目标

1. 不做期权交易建议, 只服务于 QQQ 现货/ETF 买点判断。
2. 不做分钟级或 tick 级盘中择时, MVP 仍以日频周频月频任务为主。
3. 不引入运行时自由搜索, 不允许 LLM 在执行期自行寻找数据源。
4. 不在 MVP 直接支持自动下单。
5. 使用免费数据源
