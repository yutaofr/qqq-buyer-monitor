## 1. 概述

### 1.1 文档目的

本 SDT 定义 v8.0 的完整测试策略。涵盖单元测试、集成测试、回归测试和回测验证四个层次，确保 SRD 中 19 项验收标准（AC-1 ～ AC-19）和 5 项非功能要求（NFR-1 ～ NFR-5）全部有可执行的测试覆盖。

### 1.2 v8.0 测试范围

v8.0 改造涉及三刀手术，测试要覆盖以下增量改动和回归保护：

| 改造主题 | 新增测试 | 回归保护 |
| :--- | :--- | :--- |
| 系统身份确立：删除金额计算，新增 `BetaRecommendation` | 单元 + 集成 | v7 `test_execution_policy.py` 适配 |
| Tier-0 硬约束接入 `Risk Controller` | 单元 + 集成 | v7 `test_risk_controller.py` 回归 |
| Tier-0 软约束接入 `Deployment Controller` | 单元 + 集成 | v7 `test_deployment_controller.py` 回归 |
| 存量/增量推荐解耦 | 单元 + 集成 | 移除 cash-precheck 依赖 |
| `AllocationState` 依赖切断 | 单元 | v7 `test_allocation_search_v6_4.py` 回归 |
| 两段式输出重构 | 集成 + 输出快照 | v7 CLI 回归 |

### 1.3 测试层次与职责

```
┌─────────────────────────────────────────────┐
│  L4: 回测验证 (Backtest Validation)           │  ← 历史窗口保真度
├─────────────────────────────────────────────┤
│  L3: 集成测试 (Integration)                   │  ← 端到端信号链
├─────────────────────────────────────────────┤
│  L2: 组件间契约 (Contract)                     │  ← 接口兼容性
├─────────────────────────────────────────────┤
│  L1: 单元测试 (Unit)                          │  ← 单模块逻辑
└─────────────────────────────────────────────┘
```

### 1.4 执行方式

所有测试在 Docker 容器内执行：

```bash
# 全量单元 + 集成测试
docker-compose run --rm test

# 回测验证
docker-compose run --rm backtest
```

## 2. 测试策略

### 2.1 分层原则

- **单元测试**: 每个被改造模块的每个公共函数都有独立测试。输入覆盖正常路径、边界值和异常路径。
- **集成测试**: 模拟完整 Tier-0 → Risk → Search → Recommend 信号链，验证端到端行为。
- **回归测试**: v7.0 的现有测试在 `tier0_regime="NEUTRAL"` 默认值下必须全部绿灯。
- **回测验证**: 在历史数据上验证 Tier-0 软约束对左侧入场机会的保留效果。

### 2.2 覆盖率要求

| 层次 | 目标 |
| :--- | :--- |
| 被改造模块行覆盖率 | ≥ 90% |
| AC 验收标准覆盖率 | 100%（每条 AC 至少一个测试） |
| 分支覆盖 | 每个 Tier-0 regime 至少一个正向 + 一个反向测试 |

### 6.1 v7.0 测试回归要求

| 函数 | 新增参数 | 默认值 | 回归影响 |
| :--- | :--- | :--- | :--- |

## 11. 风险与缓解

| 风险 | 影响 | 缓解 |
| :--- | :--- | :--- |
| `build_execution_actions` 删除导致下游断裂 | 集成测试和 CLI 输出出错 | Phase 1 完成后立即跑全量测试，发现断裂后修复所有调用方 |
| `_TIER0_CAPITULATION_OVERRIDE_THRESHOLD` 阈值选择不当 | 要么太松（左侧入场过于激进）要么太紧（窗口无法打开） | 回测验证（TC-BT-001），在历史 RICH_TIGHTENING 窗口验证突破频率 |
| v7 测试适配遗漏 | 回归测试假绿 | 代码审查 + grep 全量搜索 `build_execution_actions` 调用 |
| `DeploymentState` 逻辑耦合现金余额 | 削弱推荐引擎独立性 | 移除 pre-check 逻辑，改为纯推荐模型 |
