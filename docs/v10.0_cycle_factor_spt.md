## 2. 测试分层

### 2.1 Unit

- `tests/unit/test_cycle_factor.py`
- `tests/unit/test_risk_controller.py`
- `tests/unit/test_execution_policy.py`
- `tests/unit/test_v7_signal_result.py`
- `tests/unit/test_v7_persistence.py`

- `tests/unit/test_discord_notifier.py`

核心用例：

- `CAPITULATION` 允许有限 `QLD`
- `RECOVERY` 只允许更低 `QLD`
- `MID_CYCLE` / `LATE_CYCLE` / `BUST` 锁定 `QLD`
- `Risk Controller` 正确合并 cycle ceiling
- persistence 和 runtime trace 暴露 `cycle_regime`

### 2.2 Integration

- `tests/integration/test_v7_runtime_pipeline.py`
- `tests/integration/test_v8_linear_pipeline.py`

## 3. 退出条件

- 所有新测试先失败再变绿
- 所有现有相关测试无回归
- 回测能证明：
  - `LATE_CYCLE/BUST` 没有越权 `QLD`
  - `CAPITULATION/RECOVERY` 的 `QLD` 使用是条件化而非常态化
