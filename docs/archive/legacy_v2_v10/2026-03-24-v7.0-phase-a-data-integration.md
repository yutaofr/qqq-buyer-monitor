# v7.0 Phase A Data Integration Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Replace the placeholder historical macro generator with a research-grade Class A historical data pipeline that feeds `HistoricalMacroSeeder`, `certifier`, and `backtest` using the same audited schema.

**Architecture:** Phase A is intentionally narrow. It only closes the historical evidence chain for Class A features that drive the `Risk Controller`: `credit_spread`, `credit_acceleration`, `real_yield`, `net_liquidity`, `liquidity_roc`, and `funding_stress`. The implementation adds a canonical research dataset schema, deterministic builders from primary sources, strict validation, and fail-fast behavior when required research data is missing.

**Tech Stack:** Python 3, pandas, SQLite, existing FRED/collector modules, pytest

---

## Scope

In scope:
- Historical dataset schema for Class A research inputs
- Builder scripts for historical research data
- Validation and coverage reporting
- `HistoricalMacroSeeder` upgrade to canonical schema
- `backtest` and `certifier` fail-fast requirements for missing critical evidence

Out of scope:
- Class B historical reconstruction (`VIX`, breadth, Fear & Greed, options walls)
- New risk logic thresholds
- Candidate search redesign
- Full research report regeneration beyond smoke verification

## Canonical Dataset Contract

Create one canonical historical research dataset for backtest/certification:

Required columns:
- `observation_date`
- `effective_date`
- `credit_spread_bps`
- `credit_acceleration_pct_10d`
- `real_yield_10y_pct`
- `net_liquidity_usd_bn`
- `liquidity_roc_pct_4w`
- `funding_stress_flag`
- `source_credit_spread`
- `source_real_yield`
- `source_net_liquidity`
- `source_funding_stress`
- `build_version`

Rules:
- `observation_date` is the source date.
- `effective_date` is the first trading day when the feature is allowed to be visible in research.
- No placeholder constants in the canonical file.
- Missing source data stays missing; builder may derive secondary fields only from prior observed values.
- `credit_acceleration_pct_10d` and `liquidity_roc_pct_4w` must be deterministic derivations from canonical base series, not manually injected.
- `funding_stress_flag` must be traceable to a documented proxy rule, not a hand-labeled crisis window.

## Source Policy

Primary sources only:
- `credit_spread_bps`: FRED `BAMLH0A0HYM2`
- `real_yield_10y_pct`: FRED `DFII10`
- `net_liquidity_usd_bn`: derived from FRED `WALCL`, `WDTGAL`, `RRPONTSYD`
- `funding_stress_flag`: derived from FRED `NFCI` and optionally `CPFF`

Fallback policy:
- Runtime collectors may keep live fallbacks.
- Research builders must not use runtime fallback proxies such as HYG or Treasury inversion.
- If a primary historical source is unavailable, builder fails with explicit coverage diagnostics.

## Implementation Notes

- Prefer adding small pure helpers to `src/research/historical_macro_builder.py` over embedding transformation logic in scripts.
- Keep runtime collectors permissive; keep research builders strict.
- Do not route research builders through live fallback code paths.
- Use source-specific columns so any future drift can be audited from the dataset itself.
- Keep all date handling timezone-naive at dataset build time unless a source forces timezone-aware datetimes.

## Acceptance Bar For Phase A

Phase A is complete only if all of the following are true:
- `HistoricalMacroSeeder` consumes canonical schema, not crisis-window hand labels.
- `run_backtest()` refuses to run without canonical historical macro evidence.
- `certifier` refuses to grant `CERTIFIED` without audited external evidence.
- Full test suite passes.
- Backtest runs successfully with a built dataset.
- Result analysis explicitly states whether real historical data changed MDD / beta fidelity / candidate certification outcomes.
