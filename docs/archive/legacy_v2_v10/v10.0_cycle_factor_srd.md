## 1. 目标

v10.0 的目标不是再发明一个新的配比表，而是把系统从 “`Tier-0 + Risk + Deployment`” 的风险控制器，升级为 “`Tier-0 + Cycle + Risk + Candidate + Advisory + Deployment`” 的周期驱动架构。

核心结论：

1. `QQQ` 是可穿越周期的核心 beta。
2. `QLD` 不是常态资产，而是只能在极端左侧周期窗口有限使用的战术放大器。
3. 周期判定必须优先于风险状态判定。
4. 系统主 contract 仍是 `target_beta`，但 `QLD` 是否可用与可用上限，必须由周期层单独决定。

## 2. 第一性原理

### 2.1 风险补偿

当 `ERP` 过低时，市场给权益风险的补偿不足。此时继续提高 beta 或保留杠杆，不是进攻，而是低赔率下注。

### 2.2 流动性约束

信用利差和其加速度代表真实资金成本。`credit_spread` 高且 `credit_acceleration` 上行，意味着科技成长资产的贴现率与融资环境同步恶化。

### 2.3 内部腐烂

`breadth` 与 `price_vs_ma200` 不是锦上添花的软信号。它们回答的是：

- 价格是否只是少数权重股托住的假强势
- 趋势资金是否已经从风险资产撤出

### 2.4 杠杆使用边界

`QLD` 只能在满足以下条件时进入合法域：

- 高 ERP，说明估值赔率改善
- 高 credit spread 但不再继续恶化，说明清算接近尾声
- 广度和趋势处于极弱区，说明市场已出现真正恐慌

除此以外，`QLD` 默认锁定。

## 3. v10.0 周期层

新增 `CycleDecision`，输出：

- `cycle_regime`
- `target_exposure_ceiling`
- `qld_share_ceiling`
- `reasons`

### 3.1 周期状态

| Cycle Regime | 含义 | target_exposure_ceiling | qld_share_ceiling |
| :--- | :--- | ---: | ---: |
| `CAPITULATION` | 高估值赔率、高利差、极端恐慌、信用停止恶化 | `1.20` | `0.25` |
| `RECOVERY` | 信用解冻、赔率仍在、趋势开始修复 | `1.00` | `0.10` |
| `MID_CYCLE` | 正常扩张，不给杠杆特权 | `0.90` | `0.00` |
| `LATE_CYCLE` | 低 ERP + 信用收紧 + 广度/趋势变差 | `0.80` | `0.00` |
| `BUST` | 利差失控或左尾共振已形成 | `0.50` | `0.00` |
| `UNQUALIFIED` | 核心周期数据不足，防守降级 | `0.80` | `0.00` |

### 3.2 周期判定核心因子

- `erp`
- `credit_spread`
- `credit_acceleration`
- `breadth`
- `price_vs_ma200`
- `rolling_drawdown`
- `liquidity_roc`
- `funding_stress`

## 4. 决策链

v10.0 生产决策链：

`Tier-0 Structural Regime -> Cycle Decision -> Risk Controller -> Candidate Selection -> Advisory Rebalance -> Deployment`

约束顺序：

1. `Tier-0` 继续保留，用于兼容 v9 历史输出与 deployment 软约束。
2. `CycleDecision` 决定 `QLD` 是否可用，以及 target beta 的上限框架。
3. `Risk Controller` 将周期 ceiling、回撤预算、信用/流动性压力合并成最终 `RiskDecision`。
4. `Candidate Selection` 只在最终 `target_exposure_ceiling + qld_share_ceiling` 的合法域内选候选。
5. `Deployment Controller` 只决定新增现金的 `QQQ` 节奏，不授予 `QLD`。

## 5. 数据与研究约束

### 5.1 live

- `ERP`: 仍使用系统现有 `earnings_yield - real_yield` 代理
- `breadth`: 使用现有 `pct_above_50d` 代理
- `trend`: 新增 `price_vs_ma200`

### 5.2 backtest

- `credit_spread / credit_acceleration / liquidity_roc / funding_stress`: 继续使用 canonical macro seeder
- `breadth`: 使用与 live 一致的 QQQ 50 日偏离代理重建
- `trend`: 由历史价格滚动重建 `price_vs_ma200`

## 6. 验收标准

- `AC-V10-01`: 新增 `CycleDecision` 领域模型与生产判定器
- `AC-V10-02`: `QLD` 在 `LATE_CYCLE`、`BUST`、`MID_CYCLE` 中默认锁定
- `AC-V10-03`: 只有 `CAPITULATION` 与 `RECOVERY` 允许 `QLD`
- `AC-V10-04`: live 与 backtest 复用相同周期判定逻辑
- `AC-V10-05`: runtime trace / web / discord 必须显示 `cycle_regime`
- `AC-V10-06`: backtest 输出必须包含 `cycle_regime`
- `AC-V10-07`: 所有新增行为必须通过 TDD 红绿循环验证
- `AC-V10-08`: unit / integration / targeted lint / portfolio backtest 全绿

## 7. 非目标

- 不引入新的外部付费数据源
- 不把 Deployment 重新变回仓位控制器
- 不在 runtime 临时搜索未认证配比
- 不把 `QLD` 恢复为“默认可持有”的中性资产
