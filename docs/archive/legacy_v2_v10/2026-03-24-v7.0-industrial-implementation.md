# v7.0 Industrial Allocation Refactor Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.
>
> **Archive note:** this plan documents the pre-v8.0 implementation path. The current main branch has moved to the v8.1 linear pipeline, so the code samples below are retained only for historical reference.

**Goal:** Replace the v6.x live mini-backtest allocator with a production-grade v7.0 dual-controller architecture built around offline-certified candidates and deterministic runtime selection.

**Architecture:** v7.0 splits runtime decision-making into `Risk Controller` and `Deployment Controller`, with a versioned `CandidateRegistry` generated offline by research certification. Runtime no longer runs nested candidate backtests; it loads certified candidates, decides risk and deployment states, chooses a target candidate, and executes only when state changes or drift bands are breached.

**Tech Stack:** Python 3.12+, pandas, numpy, dataclasses, SQLite, pytest
