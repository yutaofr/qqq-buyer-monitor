# QQQ-Monitor v8.2 Web 发布架构终定案 (Final SRD)

## 1. 架构核心公理 (Architectural Axioms)
*   **指令覆盖数据 (Command Over Data)**: 后端不仅下发业务数据，更直接下发前端的生命周期指令。前端剥离一切业务计算权，降级为纯粹的状态机执行器。
*   **算力劣化假定 (Degradation Assumption)**: 架构层必须承认并吸收公有云免费基建的劣根性。一切过期时间的设定，必须预留至少 4 小时的 CI/CD 调度抖动（Jitter Buffer）。
*   **物理对象覆写 (Physical Overwrite)**: 彻底摒弃版本控制思维发布数据。通过 Vercel Blob 进行无历史记录的单文件强制 PUT 覆写，确保边缘节点的绝对一致性。
*   **带外主动告警 (Out-of-band Alerting)**: 系统的静默失败不可接受。发布流水线必须具备在致命错误发生时，绕过 Web 侧直接向系统管理员推送警报的能力。

## 2. 状态机驱动模型 (Three-State Engine)

前端 UI 仅由以下三种状态驱动。任何偏离预期的数据结构或网络超时，强制降维至 `STALE`。

| 状态 | 触发条件 (后端逻辑) | UI 表现 (前端执行) | 语义 |
| :--- | :--- | :--- | :--- |
| **`ACTIVE`** | 交易时段，且数据链路完整。 | **绿色 (Live)** | 信号实时，市场运行中。 |
| **`FROZEN`** | 休市/周末/节假日。 | **灰色/蓝色 (Snapshot)** | 信号有效，但市场已冷冻。 |
| **`STALE`** | `Now > expires_at_utc` 或 Fetch 失败。 | **红色 + 灰度遮罩 (Alert)** | 致命错误，流水线中断。 |

## 3. 数据契约 (The Immutable Contract)

### 3.1 离散化数据协议 (JSON Schema)
输出必须严格执行“降维映射”，严禁暴露浮点数权重与具体账户规模。

```json
{
  "meta": {
    "version": "v8.2",
    "calculated_at_utc": "2026-03-27T19:00:00Z",
    "expires_at_utc": "2026-03-28T03:00:00Z", 
    "market_state": "ACTIVE"
  },
  "signal": {
    "regime": "STRESS",
    "exposure_band": "40-50%",
    "action_directive": "HOLD",
    "fidelity": "HIGH"
  }
}
```
*   **`expires_at_utc` 计算逻辑**: `预期下一次运行时间 + 4hr (Jitter Buffer)`。

## 4. 分发与存储规约 (Distribution Layer)

### 4.1 存储选型: Vercel Blob
*   **路径**: `status.json` (固定文件名)。
*   **策略**: 绝对覆写模式。
*   **鉴权**: 依赖 GitHub Secrets 中的 `BLOB_READ_WRITE_TOKEN`。

### 4.2 边缘缓存防御 (Edge Defense)
上传时必须注入以下 HTTP Header 以保护存储源站：
`Cache-Control: public, max-age=3600, s-maxage=3600, stale-while-revalidate=600`

## 5. 零智商呈现层 (Zero-IQ Frontend)

展示层由单个 `index.html` 构成，禁止引入任何重型框架或构建链。

### 5.1 核心逻辑 (Vanilla JS)
1.  **Fetch**: 请求 `status.json`，Header 设为 `cache: "no-store"` (仅针对浏览器端，穿透由 CDN 处理)。
2.  **Safety Check**: `if (new Date() > new Date(data.meta.expires_at_utc)) return renderStale();`
3.  **Render**: 根据 `market_state` 渲染对应的颜色和文案。

## 6. 带外告警逻辑 (Out-of-band Alerting)

### 6.1 流水线监控 (GitHub Actions)
在 `deploy-web.yml` 中强制配置失败钩子：
```yaml
- name: Notify on Failure
  if: failure()
  run: |
    curl -X POST -H "Content-Type: application/json" \
         -d '{"text": "🚨 QQQ-Monitor Pipeline Failed: Check GitHub Actions"}' \
         ${{ secrets.ALERT_WEBHOOK_URL }}
```

## 7. 本地共存与环境隔离
*   **环境变量驱动**: 仅在 `EXPORT_WEB=1` 时激活 `WebExporter`。
*   **零耦合**: 该系统的引入不得修改 `src/engine/` 下任何算法逻辑。
