# SRD: 个人资产配置与 30% 回撤预算系统 (v6.4)

## 1. 概述
本系统面向个人长期投资者，而非主权基金或养老基金。它在现有 QQQ 监控与战略配置框架之上，新增一个以“**30% 最大可承受回撤**”为硬约束的个人配置层。

系统目标不是追求最低回撤，也不是追求最激进收益，而是在**可接受波动范围内尽量提高长期收益**，并避免在极端回撤中出现过度防守、错失长期复利的情况。

## 2. 核心设计原则
- **收益优先，但受风险预算约束**：策略必须优先服务长期复利，不允许为了极端防守而长期持有过高现金。
- **30% 回撤上限**：系统的设计目标是将组合峰值回撤控制在 30% 以内。超过该阈值时，系统必须进入强防御模式。
- **避免过度保守**：只有在宏观、流动性或价格结构共同恶化时，才允许明显提高现金占比。
- **少交易、重纪律**：允许每日风险审计，但实际执行应尽量降低不必要的换手。
- **单一事实源**：现状快照、目标配置、回测审计三者必须口径一致，禁止同一指标存在多套解释。
- **透明可追溯**：每一次配置变化都必须能追溯到明确的状态、证据和阈值。

## 3. 个人风险预算

### 3.1 风险目标
- **硬约束**：组合最大回撤不得长期突破 30% 设计预算。
- **软警戒**：回撤达到 20% 时，系统进入警戒观察。
- **强防御**：回撤达到 25% 时，系统必须降低风险敞口。
- **强制防御**：回撤达到 30% 时，系统必须进入资本保护模式。

### 3.2 风险偏好定位
本 SRD 不要求机构级“极低回撤”，而要求：
- 保留足够的权益暴露，避免现金拖累长期复利。
- 在回撤和风险信号恶化时，能够快速降风险。
- 在风险恢复后，能够尽快恢复权益暴露，不长期停留在高现金状态。

## 4. 目标配置模型矩阵 (Personal Target Bands)

系统基于 `AllocationState` 映射以下 **TargetAllocationState** 配置带。比值顺序固定为 **QQQ:QLD:Cash**。实现时可复用现有枚举名，但对外解释应遵循本节的个人投资者语义。

说明：本节定义的是**可接受比例带**，不是死板单点值。最终精确比值应由回测在约束范围内自动选出，并优先满足风险预算，其次再比较长期收益与回撤质量。也就是说：
- **SRD 定规则**
- **ADD 定实现**
- **回测选最优比值**

| AllocationState | 触发语义 | 配置比值 (QQQ:QLD:Cash) | 约化百分比 | Target Beta | 说明 |
| :--- | :--- | :---: | :---: | :---: | :--- |
| **FAST_ACCUMULATE** | 最强势阶段 / 风险偏好最高 | **4:4:2** 或 **4:3:3** | 40/40/20 或 40/30/30 | **1.20 ~ 1.10** | 强势阶段允许少量到中等 QLD，现金维持在 20%~30% |
| **BASE_DCA** | 中性基准 / 默认持有 | **5:2:3** 或 **6:1:3** 或 **7:1:2** | 50/20/30 或 60/10/30 或 70/10/20 | **0.90 ~ 0.80** | 个人投资者的默认增长仓位，兼作平滑过渡中枢 |
| **SLOW_ACCUMULATE** | 温和回撤 / 震荡后修复 | **6:0:4** 或 **7:1:2** | 60/0/40 或 70/10/20 | **0.70 ~ 0.80** | 不追求高杠杆，优先保留中等权益暴露 |
| **WATCH_DEFENSE** | 信号转弱 / 风险上升 | **7:0:3** | 70/0/30 | **0.70** | 进入观察期，停止追高，保留核心权益仓 |
| **DELEVERAGE** | 回撤扩大 / 信贷或流动性恶化 | **6:0:4** | 60/0/40 | **0.60** | 降低权益敞口，但不完全离场 |
| **CASH_FLIGHT** | 三重共振 / 接近或突破 30% 回撤预算 | **6:0:4** 或 **7:0:3** | 60/0/40 或 70/0/30 | **0.60 ~ 0.70** | 进入资本保护，但不常态化极端空仓 |

### 4.1 约束规则
- 任何防御状态下，`Target QLD% = 0`。
- `FAST_ACCUMULATE` 是唯一允许较高 QLD 比重的状态，且不得超过 `4:4:2` 这一强势上限。
- `BASE_DCA` 是系统常态中枢，应作为大多数市场环境下的默认配置；`613`、`712` 属于中性附近的合法配置。
- `WATCH_DEFENSE`、`DELEVERAGE`、`CASH_FLIGHT` 必须落在 `7:0:3` 到 `6:0:4` 的保守带内，并按风险信号平滑切换。
- 系统不得从 `4:4:2` 直接跳到 `7:0:3` 或 `6:0:4`，必须通过中间带完成过渡。
- 若风险状态恢复，系统应优先回到 `BASE_DCA`，再逐步回到 `FAST_ACCUMULATE`，避免一次性反向跳变。
- 每个状态的最终精确比值必须在本节给定比例带内由回测选出，不得超出约束带后再以“收益更高”为理由回填。

## 5. 技术协议规范

### 5.1 输入归一化与安全性协议
- **读取流**：解析 `CASH_LEVEL`, `QQQ_LEVEL`, `QLD_LEVEL`，缺失时默认 `0.0`。
- **合法性截断**：所有负值强制修正为 `0.0`。
- **分母安全检查**：计算 `Sum = CASH + QQQ + QLD`。
- **降级路径**：
  - 若 `Sum <= 0` 或包含 `NaN/Inf`，强制返回 `Reality = [1.0, 0.0, 0.0]`。
  - 否则按比例归一化。

### 5.2 价格与回测协议
- **QQQ** 使用历史收盘价。
- **QLD** 作为 2x 近似杠杆资产，必须显式考虑日度收益放大与费率损耗。
- **回测单位** 使用 `Cash + QQQ + QLD` 的净值核算。
- **再平衡原则**：允许每日审计，但执行层应优先使用“状态变化 + 偏离带宽”触发，而非无条件高频交易。

### 5.3 风险执行协议
- 当组合回撤达到 20% 时，必须进入观察态，配置应从 `4:4:2` 逐步回落到 `5:2:3`、`6:1:3` 或 `7:1:2`。
- 当组合回撤达到 25% 时，必须降低杠杆/降低风险敞口，配置应向 `6:0:4`、`7:1:2` 过渡。
- 当组合回撤达到 30% 时，必须强制切换到资本保护模式，配置应位于 `7:0:3` 或 `6:0:4` 的保守带。
- 当风险信号恢复且回撤回落后，允许逐步回到 `BASE_DCA` 或 `FAST_ACCUMULATE`，但必须保持平滑过渡，不允许直接跳档。
- 在所有满足风险约束的候选方案中，回测应优先选择长期 CAGR 更高、最大回撤更低、平均 Beta 偏差更小、换手率更低的配置。

## 6. 输出与审计要求
- 系统必须输出当前现状快照 (`CurrentPortfolioState`) 与目标配置 (`TargetAllocationState`)。
- 系统必须保留 `effective_exposure` 作为审计指标，但不得将其伪装成真实贝塔。
- 回测结果必须展示：
  - 最大回撤
  - 平均实现贝塔偏差
  - 防御状态覆盖时长
  - 平均持仓成本
  - 资金部署效率
- 所有审计结果必须可通过 JSON / DB / 报告回放。

## 7. 验收标准 (Acceptance Criteria)

- **AC-1**: 在全零输入下，系统不报错，且 `Reality` 必须回退为 100% 现金。
- **AC-2**: 在 `WATCH_DEFENSE`、`DELEVERAGE`、`CASH_FLIGHT` 状态下，`Target QLD%` 必须为 0。
- **AC-3**: 回测中每一时刻必须满足 `NAV = Cash + QQQ + QLD`，浮点误差应小于 `1e-4`。
- **AC-4**: 历史回测中，所有有效区间的 `Mean Absolute Beta Deviation` 必须 `<= 0.05`。
- **AC-5**: 全量历史回测的最大回撤必须控制在 `30%` 以内，或在超过预算时明确触发 `CASH_FLIGHT`。
- **AC-6**: 全量历史回测中，长期 CAGR 不得低于基准 DCA 的 `90%`，以避免策略过度保守。
- **AC-7**: 所有关键审计字段必须可在 JSON / SQLite 中完整回放，包括 `interval_beta_audit` 与 `target_allocation`。

## 8. 非目标
- 不追求机构级的极低回撤。
- 不追求市场中性或绝对收益。
- 不要求高频交易。
- 不要求对每一次短期噪声都立即反应。

## 9. 说明
本 SRD 是面向个人投资者的风险预算文档。它的默认哲学是：

> 不要为了绝对安全而牺牲长期复利，也不要为了长期复利而忽略自己真实能承受的回撤边界。
