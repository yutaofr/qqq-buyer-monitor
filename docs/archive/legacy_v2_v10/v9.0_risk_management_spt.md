# SPT: v9.0 风险管理重构测试计划

> 版本: v9.0
> 状态: Approved for implementation
> 日期: 2026-03-28
> 对应 SRD: `docs/v9.0_risk_management_matrix.md`
> 对应 ADD: `docs/v9.0_risk_management_add.md`

## 1. 测试目标

验证 v9.0 满足以下核心要求：

- `target_beta` 仍是系统主 contract
- `QLD` 可以在任意 beta 下参与，但必须 obey `qld_share_ceiling`
- `RISK_REDUCED` 与 `RISK_DEFENSE` 均允许有限 `QLD`
- `Deployment Controller` 只负责新增现金买入 `QQQ` 的节奏
- live 与 backtest 的决策路径保持一致

## 2. 测试分层

### 2.1 单元测试

测试文件：

- `tests/unit/test_risk_controller.py`
- `tests/unit/test_allocation_search_v8.py`
- `tests/unit/test_candidate_registry.py`
- `tests/unit/test_execution_policy.py`

核心用例：

#### TC-V9-001 风险状态输出 `qld_share_ceiling`

- 验证：
  - `RISK_ON` / `RISK_NEUTRAL` / `RISK_REDUCED` / `RISK_DEFENSE` / `RISK_EXIT`
  - 每个分支都带 `qld_share_ceiling`

#### TC-V9-002 ceiling 随 stress 单调收缩

- 预期：
  - `RISK_ON >= RISK_NEUTRAL >= RISK_REDUCED >= RISK_DEFENSE >= RISK_EXIT`

#### TC-V9-003 候选过滤同时 obey beta 与 `QLD` ceiling

- 前置条件：
  - 两个候选 beta 相同
  - 一个 `qld_pct` 在 ceiling 内
  - 一个 `qld_pct` 超 ceiling
- 预期：
  - 超 ceiling 候选被拒绝

#### TC-V9-004 floor fallback 也 obey `QLD` ceiling

- 前置条件：
  - scoped candidates 不合规
  - registry 有 `0.5 beta` floor 候选
- 预期：
  - fallback 只能返回 `qld_pct <= qld_share_ceiling` 的候选

#### TC-V9-005 `target_allocation_from_beta()` 支持 sub-1 beta 的有限 `QLD`

- 预期：
  - `target_allocation_from_beta(0.80, qld_share_ceiling=0.10)` 生成合法 `613`
  - `target_allocation_from_beta(0.90, qld_share_ceiling=0.20)` 生成合法 `523`
  - `target_allocation_from_beta(1.00, qld_share_ceiling=0.20)` 生成合法 `622`

#### TC-V9-006 `qld_share_ceiling=0.0` 时退化为纯 `QQQ + Cash`

#### TC-V9-007 registry fixture 含 limited-`QLD` 候选

- `RISK_REDUCED` 至少 1 个
- `RISK_DEFENSE` 至少 1 个
- `RISK_EXIT` 不得有 `QLD > 0`
