# ADD: 个人资产配置与 30% 回撤预算实现方案 (v6.4)

## 1. 架构目标
实现一套面向个人投资者的战略资产配置引擎：在满足 30% 回撤预算和 Beta 审计约束的前提下，允许系统在多个候选比例带之间进行回测搜索，并自动选出最优的 `QQQ:QLD:Cash` 比值。

本 ADD 的核心原则是：
- **SRD 定规则**：定义允许的比例带、风险预算和约束边界。
- **ADD 定实现**：定义如何枚举、回测、筛选和落盘候选配置。
- **回测选最优解**：最终精确比值由历史回测结果决定，而不是写死在配置表里。

## 2. 核心组件设计

### 2.1 候选比例层 (`src/engine/allocation_search.py`)
- 实现 `TargetBandCandidate` 与 `AllocationSearchResult`。
- 输入来自 SRD 中定义的比例带，例如：
  - `FAST_ACCUMULATE`: `442` / `433`
  - `BASE_DCA`: `523` / `613` / `712`
  - `SLOW_ACCUMULATE`: `604` / `712`
  - `WATCH_DEFENSE`: `703`
  - `DELEVERAGE`: `604`
  - `CASH_FLIGHT`: `703` / `604`
- 搜索器不得越界生成比例，只能在 SRD 允许的带内做离散枚举或插值采样。

### 2.2 回测评分层 (`src/backtest.py`)
- 对每个候选比例执行同一套历史回测。
- 对每个候选输出以下指标：
  - `max_drawdown`
  - `mean_interval_beta_deviation`
  - `cagr`
  - `turnover`
  - `defense_coverage`
  - `nav_integrity`
- 回测模型必须沿用当前多资产单位制仿真，不得退回单资产近似。

### 2.3 策略选择器 (`src/engine/aggregator.py`)
- 为每个 `AllocationState` 选出一个最优 `TargetAllocationState`。
- 选择规则采用“硬约束优先、收益目标次之”的顺序：
  1. 过滤掉违反 `AC-3`、`AC-4`、`AC-5` 的候选。
  2. 在剩余候选中，优先选择 `CAGR` 更高者。
  3. 若 `CAGR` 相近，则选择 `max_drawdown` 更低者。
  4. 若仍相近，则选择 `mean_interval_beta_deviation` 更小者。
  5. 最后用更低 `turnover` 作为平局裁决。

## 3. 最优比值选择协议

### 3.1 强势阶段
- `FAST_ACCUMULATE` 允许 `442` 和 `433` 两个候选。
- 回测应在满足风险约束前提下优先保留更高长期收益的方案。
- 若 `442` 与 `433` 收益接近，则优先选择更低换手的一项。

### 3.2 中性阶段
- `BASE_DCA` 允许 `523`、`613`、`712`。
- `613` 与 `712` 视为中性到偏进取的合法过渡态。
- 系统不得把中性阶段固化为单一固定点，必须允许回测根据历史环境选出最优中心值。

### 3.3 保守阶段
- `WATCH_DEFENSE`、`DELEVERAGE`、`CASH_FLIGHT` 必须停留在 SRD 定义的保守带。
- 不允许在防御状态下为了提高回测收益而引入更高 QLD 或突破现金下限。
- 若多个候选均满足约束，优先选择风险更低而非收益更高的方案。

## 4. 持久化与审计

### 4.1 结果落盘
- 最终选中的 `TargetAllocationState` 必须写入 `SignalResult`。
- `current_portfolio`、`target_allocation`、`effective_exposure`、`interval_beta_audit` 必须进入 JSON / SQLite。
- 历史回放必须能够重建“当前状态 -> 候选搜索 -> 最终选择”的全过程。

### 4.2 报告输出
- 报告中必须同时展示：
  - 搜索到的候选比例
  - 被淘汰的原因
  - 最终选定的最优比例
  - 对应的回测指标
- 报告不得只给出最终结果而不展示选择依据。

## 5. 验收路径 (Acceptance Path)

- **AC-1**: 在同一 `AllocationState` 下，回测搜索能从 SRD 允许的候选中返回一个且仅一个最优比例。
- **AC-2**: 对任何防御状态，搜索结果的 `Target QLD%` 必须为 0。
- **AC-3**: 所有候选回测都满足 `NAV = Cash + QQQ + QLD`，浮点误差小于 `1e-4`。
- **AC-4**: 所有有效区间的 `Mean Absolute Beta Deviation` 必须 `<= 0.05`。
- **AC-5**: 全量历史回测的最大回撤必须控制在 `30%` 以内，或在超过预算时明确进入 `CASH_FLIGHT`。
- **AC-6**: 最终选定的中性中心点必须来自 SRD 允许集合，例如 `523`、`613` 或 `712`，不得越界。
- **AC-7**: 所有搜索与回测结果必须可通过 JSON / SQLite 完整回放。

## 6. 实现说明
- 本 ADD 不把某个单点比例写死为真理，而是把“可接受比例带”作为制度边界。
- 最终最优比值的决定权属于历史回测与风险约束共同作用的结果。
- 若未来回测环境变化，只需要重跑搜索层，不需要重写 SRD。

## 7. Claude Code 实现顺序
为了让代码实现保持可拆分、可测试、可回滚，建议按以下顺序落地：

1. **模型层**
   - 先补齐 `CurrentPortfolioState` 与 `TargetAllocationState` 的字段、序列化与反序列化。
   - 先保证输入归一化和安全回退可单测。

2. **候选搜索层**
   - 再实现 `allocation_search.py`。
   - 只允许在 SRD 定义的比例带内生成候选，不允许越界。

3. **回测评分层**
   - 再实现 `backtest.py` 的候选回测与评分。
   - 重点保证 `NAV`、`beta`、`MDD`、`CAGR` 指标可重复。

4. **策略选择层**
   - 再实现 `aggregator.py` 的候选过滤与最优解选择。
   - 必须先过硬约束，再比收益和稳定性。

5. **持久化与报告层**
   - 最后打通 JSON / SQLite / 报告输出。
   - 确保 `current_portfolio`、`target_allocation`、`interval_beta_audit` 都能回放。

6. **测试补强**
   - 最后补端到端测试。
   - 优先覆盖：全零输入、防御状态 QLD=0、AC-3、AC-4、AC-5、候选最优解选择。
