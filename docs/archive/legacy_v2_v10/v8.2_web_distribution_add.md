# Architecture Design Document (ADD) - QQQ-Monitor Web Distribution (v8.2)

## 1. 物理层架构公理 (Physical Axioms)
*   **统一 UTC 交换逻辑 (UTC-Only Exchange)**: 禁止在网络传输中使用 Naive Datetime。后端生产、JSON 存储、前端消费必须强制对齐 `ISO-8601 UTC (Z)`。
*   **日历前瞻跃迁 (Forward-Looking Calendar)**: 信号失效时间不是基于线性的运行时间，而是基于“下一个交易窗口开启时刻”的非线性跳跃，必须兼容美股“半日交易”逻辑。
*   **环境隔离栅栏 (Staging Gates)**: 生产环境 Token 具备环境感知力。非 GitHub Actions 运行环境物理隔离生产存储桶的写入权限。
*   **失效即死亡 (Fail-to-Stale)**: 前端不设“加载中”或“部分可用”状态。任何异常（网络、时区解析、格式错误）必须立即触发全屏锁定与红色警报。

## 2. 核心组件精化设计

### 2.1 生产者: `WebExporter` (Market-Aware)
*   **时区对齐协议**: 
    *   内部计算：使用 `pytz.timezone('US/Eastern')`。
    *   输出序列化：强制转为 `ISO-8601 UTC (Z)` 字符串。
*   **日历感知跃迁 (The Leap Logic)**:
    *   使用 `pandas_market_calendars` 获取精确的收盘时刻（支持 1:00 PM 与 4:00 PM）。
    *   `if (now > Today_Close)` -> `Next_Expected_Run = Next_Trading_Day_Open`。
    *   `expires_at_utc = Next_Expected_Run + 4hr (Jitter Buffer)`。
*   **环境安全开关**:
    *   `GITHUB_ACTIONS=true` -> 允许生产写入。
    *   `Local/Manual` -> 强制 Dry-run 或写入 Staging 存储桶。

### 2.2 发布流水线: `deploy-web.yml` (Concurrency Control)
*   **并发互斥**: `concurrency.group: deploy-blob` + `cancel-in-progress: true`。
*   **运行环境**: 剥离 Docker，使用 Ubuntu Runner + Pip 极简环境以获得最快启动速度。

### 2.3 存储分发: Vercel Blob (Cache Invalidation)
*   **HTTP Header 协议**: 
    `Cache-Control: public, s-maxage=3600, max-age=0, must-revalidate`
    *   **s-maxage=3600**: 边缘 CDN 节点缓存 1 小时，降低存储源站负载。
    *   **max-age=0**: 强制浏览器每次必须向 CDN 请求，确保用户能够看到最新的信号更新。

### 2.4 展示层: `index.html` (State Machine)
*   **哑巴逻辑实现**: 
    ```javascript
    try {
        const res = await fetch(URL, { cache: 'no-store' });
        const data = await res.json();
        if (new Date() > new Date(data.meta.expires_at_utc)) throw new Error("Expired");
        renderUI(data.meta.market_state, data.signal);
    } catch (e) {
        renderStaleUI("CRITICAL: SYSTEM_STALE / PIPELINE_FAIL");
    }
    ```

## 3. 故障对冲矩阵 (Mitigation Matrix)

| 场景 | 后果 | 对冲机制 |
| :--- | :--- | :--- |
| **半天交易日 (1:00 PM 收盘)** | 线性 4:00 PM 计算失效 | `pandas_market_calendars` 获取精确收盘点，自动识别提前休市。 |
| **夏/冬令时切换** | 产生 1 小时时差 | 序列化强制 UTC 对齐，将偏移量计算留给 JS 引擎。 |
| **GA 任务积压/延迟** | 导致预警误报 | 设置 4 小时失效缓冲 (Jitter Buffer)，吸收 CI/CD 排队延迟。 |
| **本地重构误写** | 覆盖生产数据 | 环境变量检测隔离，物理拦截非 CI 环境的写入 Token。 |

## 4. 告警规约 (Monitoring)
*   **带外告警**: GitHub Actions 失败时立即通过 Webhook 发送包含 Run ID 的警报。
*   **数据完整性监控**: 建议定期检查 Vercel 访问日志，监控 4xx/5xx 异常波动。
