# Backtest Plan: v10.0 周期因子架构演进

> 版本: v10.0
> 状态: Approved for execution
> 日期: 2026-03-29

## 1. 目标

证明 v10.0 的周期层不是“叙事补丁”，而是对暴露上限和 `QLD` 权限有实质影响的制度改进。

## 2. 研究假设

1. `LATE_CYCLE` / `BUST` 锁定 `QLD` 能降低左尾路径风险。
2. `CAPITULATION` / `RECOVERY` 的条件化 `QLD` 使用，优于把 `QLD` 当成全周期常态资产。
3. live 与 backtest 共用同一周期分类器后，回测解释性与生产解释性一致。

## 3. 回测设计

### 3.1 结构验证

- 检查 `daily_timeseries` 是否包含 `cycle_regime`
- 检查 `cycle_qld_share_ceiling`
- 检查 candidate 是否 obey cycle gate

### 3.2 情景验证

- `Late Cycle`: 低 ERP + 紧信用 + 弱广度
- `Bust`: 高 spread + 正 acceleration + 趋势破坏
- `Capitulation`: 高 spread + 高 ERP + 恐慌 + 信用不再恶化
- `Recovery`: spread 回落 + trend 修复 + limited `QLD`

### 3.3 结果验证

- `target_beta`
- `selected_candidate_id`
- `qld_share_ceiling`
- `turnover`
- `raw_turnover`
- `max_drawdown`
- `nav_integrity`

## 4. 通过标准

- 回测流程成功运行
- 场景断言全部通过
- 组合回测无新的 `MDD > 30%` 回归
- 输出叙事能解释每个阶段为什么允许或禁止 `QLD`

## 5. 数据前提与当前限制

- v10 周期分类要求 `erp` / `trend` / `breadth` 等输入完整。
- 当前仓库内的 `data/macro_historical_dump.csv` 仍未提供 canonical `erp_pct` / `forward_pe` 历史列。
- 因此，完整历史 acceptance 报告在很多日期会退化到 `UNQUALIFIED`，其结果主要用于验证“系统不会越权”，不应被误读为对周期设计本身的最终统计结论。
- 当前版本对周期设计的主要证明来自：
  - scenario backtests
  - runtime / backtest parity tests
  - narrative surface alignment tests
- 下一阶段研究任务是补齐 canonical ERP 历史序列，再运行全样本 v10 acceptance re-baseline。
