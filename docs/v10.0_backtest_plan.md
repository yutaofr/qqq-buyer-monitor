# Backtest Plan: v10.0 周期因子架构演进

> 版本: v10.0
> 状态: Verified
> 日期: 2026-03-29
> 对应 Scorecard: `docs/v10.0_backtest_scorecard.md`

## 1. 目标

证明 v10.0 的周期层不是“叙事补丁”，而是对暴露上限和 `QLD` 权限有实质影响的制度改进。

## 2. 研究假设

1. `LATE_CYCLE` / `BUST` 锁定 `QLD` 能降低左尾路径风险。
2. `CAPITULATION` / `RECOVERY` 的条件化 `QLD` 使用，优于把 `QLD` 当成全周期常态资产。
3. live 与 backtest 共用同一周期分类器后，回测解释性与生产解释性一致。

## 3. 回测设计

### 3.1 结构验证

- 检查 `daily_timeseries` 是否包含 `cycle_regime`
- 检查 `cycle_qld_share_ceiling`
- 检查 candidate 是否 obey cycle gate

### 3.2 情景验证

- `Late Cycle`: 低 ERP + 紧信用 + 弱广度
- `Bust`: 高 spread + 正 acceleration + 趋势破坏
- `Capitulation`: 高 spread + 高 ERP + 恐慌 + 信用不再恶化
- `Recovery`: spread 回落 + trend 修复 + limited `QLD`

### 3.3 结果验证

- `target_beta`
- `selected_candidate_id`
- `qld_share_ceiling`
- `turnover`
- `raw_turnover`
- `max_drawdown`
- `nav_integrity`

### 3.4 增量资金节奏闭环

新增独立 pacing audit，不比较 `CAGR`，只比较资金进场节奏决策。

验证对象：

- `deployment_multiplier`
- `expected_deployment_multiplier`
- `actual_deployment_cash`
- `expected_deployment_cash`

统计指标：

- `MAE`
- `RMSE`
- `error_variance`
- `correlation`
- `explained_variance`
- `within_tolerance_ratio`
- `cash_mae`
- `cash_rmse`

执行口径：

- 只统计 weekly funding checkpoint
- 非资金进场日不纳入 pacing error
- 同时保留 categorical deployment alignment 作为 overlay
