# ADD: 宏观防御架构设计 (v6.2)

## 1. 架构总览
为了实现养老/主权基金风格的防御，系统必须在原有的 `MarketData -> Engine -> Signal` 链路中，插入 **`PortfolioState` (组合状态层)**。通过防御性旁路 (Defensive Bypass) 机制，系统在信贷收缩期能够强制干预加仓节奏并执行资产再平衡。

## 2. 模块化组件升级

### 2.1 数据采集层 (Collector) - 统一口径 (SSoT)
- **`src/collector/macro_v3.py` 升级：**
    - 统一调用 `fetch_net_liquidity(series_id="WDTGAL")` 计算 ROC。
    - 实现 `fetch_credit_acceleration(window=10)`: 获取利差变化百分比趋势。
    - 实现 `fetch_funding_stress()`: 监控 `NFCI` 和 `CPFF` 融资成本阈值。

### 2.2 核心模型扩展 (Models)
- **新增 `PortfolioState` 类：**
    - 字段：`current_cash_pct`, `leverage_ratio`, `gross_exposure_pct`, `net_exposure_pct`, `core_equity_pct`, `tactical_equity_pct`。
- **扩展 `AllocationState` 枚举：**
    - `WATCH_DEFENSE`: 对应 L1 等级。
    - `DELEVERAGE`: 对应 L2 等级。
    - `CASH_FLIGHT`: 对应 L3 等级。

### 2.3 决策单子与旁路逻辑 (Engine Bypass)
- **`src/engine/aggregator.py` 升级：**
    - 引入 **`Defensive_Bypass_Manager`**。
    - **优先级旁路判定：**
        1. **L3 (CASH_FLIGHT)**: 信用利差扩张/信用恶化 + 流动性负增长 + 融资压力共振 -> 强制 `tranche = 0` + `SELL_TO_TARGET` 指令。
        2. **L2 (DELEVERAGE)**: 信用利差扩张/信用恶化 + 流动性负增长 -> 建议减持战术仓位，锁定现金比例目标（20-35%）。
        3. **L1 (WATCH_DEFENSE)**: 仅信用利差 10 日加速 > 15% -> 禁止新增风险预算，`leverage_ratio` 锁定在 1.0。

## 3. 组合状态管理机制 (Portfolio Rebalancing)
- 系统新增一个独立的评估步骤：**`_step_portfolio_alignment`**。
- 它对比 `current_cash_pct` 与对应防御等级的 `target_cash_pct`。如果现金比例低于目标水位，则在输出中生成明确的减仓指导建议。

## 4. 叙事守卫逻辑 (Narrative Guardrail Implementation)
- **`src/output/interpreter.py` 逻辑：**
    - 构造 `DefensiveVocabularyFilter`。
    - 在输出前检查 `allocation_state`。若属于防御状态，利用正则表达式过滤所有包含“抄底/黄金坑/回撤买入”等看多暗示的文案。

## 5. 验收验证
- 通过新增集成测试 `tests/integration/test_triple_confirmation.py` 验证：
    - 在信用加速但流动性 ROC 仍大于 -2% 的情况下，系统最高仅允许输出 `WATCH_DEFENSE`。
    - 在信用加速 + 流动性负增长，但融资压力尚未显现的情况下，系统最高仅允许输出 `DELEVERAGE`。
    - 仅在信用、流动性、融资压力三者共振下，才能强制输出 `CASH_FLIGHT` 这一极端避险指令。
