# ADD: v9.0 风险管理重构

> 版本: v9.0
> 状态: Approved for implementation
> 日期: 2026-03-28
> 对应 SRD: `docs/v9.0_risk_management_matrix.md`

## 1. 架构目标

v9.0 要解决的不是“再发明一套配比系统”，而是把系统真正的 product contract 与内部实现路径重新对齐：

1. 对外 contract 仍是组合级 `target_beta`。
2. `QLD` 是否参与，不由 “`target_beta > 1.0`” 这种错误规则决定，而由同 beta 下的 `qld_share_ceiling` 决定。
3. `Deployment Controller` 只负责新增现金买入 `QQQ` 的节奏，不参与存量 beta 计算，也不决定 `QLD` 是否可用。
4. live 入口与 backtest 入口必须复用同一条 `Tier-0 -> Risk -> Candidate -> Advisory -> Deployment` 决策路径。

## 2. 第一性原理约束

### 2.1 资产层约束

组合级 beta 仍定义为：

```text
portfolio_beta = qqq_pct * 1.0 + qld_pct * 2.0 + cash_pct * 0.0
```

在合法域内：

```text
qqq_pct >= 0
qld_pct >= 0
cash_pct >= 0
qqq_pct + qld_pct + cash_pct = 1
qqq_pct + 2 * qld_pct = target_beta
```

因此：

- 同一个 `target_beta` 可以对应多种 `QQQ / QLD / Cash` 实现路径；
- `QLD` 可以出现在 `target_beta <= 1.0` 的组合中；
- 系统真正需要控制的是：
  - `target_beta`
  - `qld_share_ceiling`

### 2.2 决策层约束

- `Risk Controller` 决定存量资产的硬边界：`target_exposure_ceiling` 与 `qld_share_ceiling`
- `Deployment Controller` 决定新增现金部署快慢：`DEPLOY_*`
- 新增现金默认唯一买入标的是 `QQQ`
- 新增现金不参与存量资产 beta 计算

## 3. 设计原则

- contract first: 系统主 contract 只提升 `target_beta` 与 `deployment_state`
- implementation path internal: 配比是内部实现与回测路径，不是用户强制 contract
- monotonic stress: stress 升高时，总 beta ceiling 与 `qld_share_ceiling` 必须单调收缩
- auditability: 每次候选过滤都能解释是 beta ceiling 还是 `qld_share_ceiling` 导致拒绝
- parity: live 与 backtest 的风控、候选过滤和 advisory materialization 必须一致

## 4. 架构决策

### 4.1 `RiskDecision` 扩展 `qld_share_ceiling`

文件：

- `src/models/risk.py`
- `src/engine/risk_controller.py`

新增字段：

- `qld_share_ceiling: float`

设计语义：

- `RISK_ON` / `RISK_NEUTRAL`：允许更高 `QLD` 占比
- `RISK_REDUCED`：允许有限 `QLD`
- `RISK_DEFENSE`：只允许更低的有限 `QLD`
- `RISK_EXIT`：`QLD` ceiling 为 `0.0`

v9.0 默认 ceiling 梯度：

| Risk State | target_exposure_ceiling | qld_share_ceiling |
| :--- | :--- | :--- |
| `RISK_ON` | `1.20` | `0.25` |
| `RISK_NEUTRAL` | `1.00` | `0.20` |
| `RISK_REDUCED` | `0.80~0.85` | `0.10` |
| `RISK_DEFENSE` | `0.70` | `0.05` |
| `RISK_EXIT` | `0.50` | `0.00` |

说明：

- `RISK_REDUCED` 明确允许有限 `QLD`
- `RISK_DEFENSE` 仍允许更低限度 `QLD`，但 ceiling 必须低于 `RISK_REDUCED`
- Tier-0 与 drawdown 先决定 risk state，再继承对应 ceiling

### 4.2 候选选择从单一 beta ceiling 升级为双硬约束

文件：

- `src/engine/allocation_search.py`
- `src/engine/candidate_registry.py`
- `src/engine/runtime_selector.py`

v8.2 只过滤：

- `target_effective_exposure <= max_beta_ceiling`

v9.0 新增：

- `candidate.qld_pct <= qld_share_ceiling`

结果：

- 同 beta 下，`QLD` 占比过高的候选会被拒绝
- 同 beta 下，有限 `QLD` 的候选可以合法参与
- `global beta floor fallback` 也必须同时 obey `qld_share_ceiling`

### 4.3 Advisory materialization 显式接收 `qld_share_ceiling`

文件：

- `src/engine/execution_policy.py`

`target_allocation_from_beta(...)` 不再隐含 “`beta <= 1` 就禁用 `QLD`”。

改为：

- 输入 `target_beta`
- 输入 `qld_share_ceiling`
- 在合法域内生成一个连续、可回放的内部实现路径

默认 materialization 策略：

- 在满足 `target_beta` 的前提下，尽量使用不超过 ceiling 的 `QLD`
- 若 ceiling 为 `0.0`，退化为纯 `QQQ + Cash`

这保证：

- `0.8 beta` 可以 materialize 为 `0.6 / 0.1 / 0.3`
- `0.9 beta` 可以 materialize 为 `0.5 / 0.2 / 0.3`
- `1.0 beta` 可以 materialize 为 `0.6 / 0.2 / 0.2`

### 4.4 `Deployment Controller` 语义钉死为 `QQQ-only new cash`

文件：

- `src/engine/deployment_controller.py`
- `src/main.py`
- `src/backtest.py`
- `README-CN.md`
- `docs/backtest_report.md`

要求：

- `deployment_state` 只影响新增现金的 `QQQ` 入场速度
- 不回写 `target_beta`
- 不授予 `QLD`
- 不解释为“存量加杠杆”

### 4.5 candidate registry 的定位

文件：

- `data/candidate_registry_v7.json`
- `tests/fixtures/candidate_registry_v7.json`

v9.0 继续保留 registry，但重新定义为：

- 内部认证实现路径池
- 用于 runtime 过滤、审计和回测
- 不是用户必须遵守的唯一资产配比输出

因此 registry 需要至少满足：

- `RISK_REDUCED` 存在有限 `QLD` 候选
- `RISK_DEFENSE` 存在更低 `QLD` ceiling 下的候选
- `RISK_EXIT` 只有 `QLD=0`

## 6. 验收标准

### 6.1 功能验收

- `AC-V9-01`: `RiskDecision` 必须输出 `qld_share_ceiling`
- `AC-V9-02`: `RISK_REDUCED` 下必须允许 `QLD` 有限参与，且 ceiling `> 0`
- `AC-V9-03`: `RISK_EXIT` 下 `qld_share_ceiling == 0.0`
- `AC-V9-04`: runtime candidate selection 必须同时 obey `max_beta_ceiling` 与 `qld_share_ceiling`
- `AC-V9-05`: 同 beta 下，`candidate.qld_pct > qld_share_ceiling` 必须被拒绝
- `AC-V9-06`: `target_allocation_from_beta(0.8, qld_share_ceiling=0.1)` 必须能生成合法的有限 `QLD` 路径
- `AC-V9-07`: `Deployment Controller` 不得输出、修改或推断存量 `QLD` 配比
- `AC-V9-08`: 新增现金部署在 live 与 backtest 中都只能映射为 `QQQ` 买入节奏

## 7. 风险与缓解

- 风险: `target_allocation_from_beta()` 改成 ceiling-aware 后，组合回测路径会变化
  - 缓解: 把 signal acceptance 与 portfolio backtest 都纳入回归门槛
- 风险: registry 增加 limited-`QLD` 候选后，选择排序可能漂移
  - 缓解: 用固定 fixture metrics 锁住测试期望
- 风险: 文档继续把内部比例误写成外部 contract
  - 缓解: 所有对外文档只把 `target_beta` 与 `deployment_state` 放在第一层

## 8. 四眼评审结论

- Domain expert + architect 复核结论：通过
- 通过依据：
  - 修正了 “`QLD` 只在 `beta>1`” 的硬错误
  - 修正了 “Deployment 会改变存量 beta” 的硬错误
  - 收回到 `target_beta` first 的系统 contract
