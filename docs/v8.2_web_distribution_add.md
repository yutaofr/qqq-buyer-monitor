# Architecture Design Document (ADD) - QQQ-Monitor Web Distribution (v8.2)

## 1. 系统公理 (System Axioms)
*   **日历游标感知 (Market Cursor Awareness)**: 信号的生命周期是非线性的。必须具备跨越周末与节假日的逻辑跳跃能力。
*   **边缘强缓存/浏览器零缓存 (Edge-Max / Browser-Zero)**: 强迫浏览器每次必回源（CDN），但由 CDN 边缘内存秒级回复。
*   **流水线并发绞杀 (Pipeline Concurrency Muffling)**: 物理上禁止两条流水线同时持有写入权，新任务必须无情中断旧任务。
*   **阅后即焚环境 (Stateless Runtime)**: 承认 GitHub Actions 的临时性，所有数据均通过 API 实时获取并直接 PUT，不依赖任何本地持久化状态。

## 2. 核心组件逻辑重构

### 2.1 生产者: `WebExporter` (Market-Aware)
*   **Next-Run 跃迁算法**: 
    *   使用 `pandas_market_calendars` 获取未来交易日序列。
    *   `if (now > Today_Close)` -> `Next_Expected_Run = Next_Trading_Day_Open`。
    *   `expires_at_utc = Next_Expected_Run + 4hr (Jitter Buffer)`。
*   **三态逻辑判定**:
    *   `if (Market_Open)` -> `ACTIVE` (绿色)。
    *   `if (Market_Closed && Now < expires_at_utc)` -> `FROZEN` (蓝色/灰色)。
    *   `else` -> `STALE` (由前端兜底判定)。

### 2.2 发布流水线: `deploy-web.yml` (Atomic Concurrency)
*   **并发锁配置**:
    ```yaml
    concurrency:
      group: deploy-blob
      cancel-in-progress: true
    ```
*   **部署动作**: 在 GitHub Actions Ubuntu Runner 上极简环境运行。最大化减少启动时延。

### 2.3 存储分发: Vercel Blob (Edge Cache Policy)
*   **Header 注入协议**: 
    在执行 `PUT` 时，必须强制指定：
    `Cache-Control: public, s-maxage=3600, max-age=0, must-revalidate`
    *   **s-maxage=3600**: Vercel CDN 节点缓存 1 小时。
    *   **max-age=0**: 浏览器本地缓存失效，每次刷新必须向 CDN 发起请求。
    *   **must-revalidate**: 确保浏览器必须获得服务器的确认。

### 2.4 展示层: `index.html` (State Machine)
*   **哑巴逻辑实现**: 
    ```javascript
    const now = new Date();
    const expiresAt = new Date(data.meta.expires_at_utc);
    if (now > expiresAt) {
      renderState('STALE'); // 全屏变灰 + 红色横幅
    } else {
      renderState(data.meta.market_state); // ACTIVE 或 FROZEN
    }
    ```

## 3. 故障模式与对冲 (Failure Modes & Mitigation)

| 场景 | 后果 | 对冲机制 |
| :--- | :--- | :--- |
| **周五收盘后** | 数据不再更新 | `WebExporter` 将过期时间设为下周一开盘后，网页保持 `FROZEN` (蓝色) 状态。 |
| **GA 任务堆积** | 两个任务并发运行 | `concurrency` 锁强制杀掉旧任务，保留最新任务，防止脏写。 |
| **用户浏览器缓存** | 无法看到最新信号 | `max-age=0` 强制浏览器在每次刷新时重新拉取 CDN 缓存。 |
| **API 调用失败** | JSON 未更新 | GA 发出 `if: failure()` Webhook 告警至开发者。 |

## 4. Alpha 保护与安全 (Security & IP)
*   **离散化映射**: 仅输出 `Exposure Band (e.g., 60-70%)`，抹除一切浮点数权重和触发阈值。
*   **Token 隔离**: 写入 Token 仅存在于 GA 环境，Web 展示层无权限接触存储源站。
