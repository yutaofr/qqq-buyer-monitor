# QQQ Buy-Signal Monitor v2.0 - 技术背离与宏观过滤特性需求文档 (PRD)

## 1. 业务背景与问题定义

**现状与挑战**：
目前的 QQQ Buy-Signal Monitor (v1.0) 采用“现货/情绪 (Tier 1) + 期权支撑确认 (Tier 2)”的双层架构。它在捕捉“绝对跌幅/恐慌极值”时非常有效，并且具备优秀的防守过滤能力。
然而，v1.0 系统的所有指标（MA200 偏离、回撤、VIX、涨跌比例）都是基于“静态绝对数值”的单点评估。在极其恶劣的长时间熊市（或流动性危机）的尾部，绝对数值可能一直处于超卖区，导致寻找长线左侧买点时精确度受限。

**核心目标**：
在 v1.0 的基础上引入**时间维度的相对动能分析**。通过捕捉市场内蕴动能与表象价格之间的**“底背离（Divergence）”**，进一步提升长线定投底仓的胜率和买点质量。同时，引入极其罕见的**宏观流动性熔断（Regime Filter）**，作为防备系统性崩溃的最后一道防线。

---

## 2. 核心功能设计 (Features Spec)

### 2.1 引入全新维度的加分项：底背离 (Divergence Bonus)

**系统定位**：
不作为一票否决项，而是作为 Tier-1 的**额外加分项（Bonus Points）**。当市场出现极限恐慌引发价格新低，但内部动能拒绝新低时，给予极高权重的加分，直接促成 `TRIGGERED` 状态。

#### Feature 1: 价格与市场广度背离 (Price-Breadth Divergence)
*   **适用场景**：指数被少数几只巨头权重股带崩创新低，但大部分中小盘股已经跌不动甚至开始反弹。这通常是“市场底”早于“指数底”的标志。
*   **计算逻辑**：
    1. 判定 QQQ 价格创出过去 N 天（如 60 天）的波段新低。
    2. 对比同期（N天内）市场广度指标：此时的 `^ADD`（涨跌比）或 `Pct_Above_50D`（在 50 日均线以上的股票比例）的值，高于上一次价格低点时的广度值。
*   **执行动作**：触发【广度底背离】信号，Tier-1 额外奖励 +15 或 +20 分。

#### Feature 2: 价格与波动率恐慌背离 (Price-VIX Divergence)
*   **适用场景**：现货价格持续下跌创新低，但期权市场的恐慌情绪正在钝化，VIX 不再创新高。说明专业机构的防护仓位已经打满，抛压衰竭。
*   **计算逻辑**：
    1. 判定 QQQ 价格创出过去 N 天波段新低。
    2. 对比同期的 VIX 最高值，此时的 VIX 值**明显低于**上一波大跌时的 VIX 峰值。
*   **执行动作**：触发【恐慌底背离】信号，Tier-1 额外奖励 +10 分。

#### Feature 3: 传统技术动能背离 (Price-RSI / MACD Divergence)
*   **适用场景**：基于周线或长周期日线的纯动能衰竭指标，适用于标准的左侧波段确认。
*   **计算逻辑**：
    1. 计算 14 日 RSI（相对强弱指标）。
    2. 判定 QQQ 价格创新低，但对应的 RSI 指标形成“更高低点 (Higher Low)”。
*   **执行动作**：由于传统技术指标存在钝化风险，此信号作为辅助，触发【RSI底背离】，Tier-1 额外奖励 +5 或 +10 分。

---

### 2.2 引入全新维度的防御层：宏观流动性熔断 (Macro Regime Filter / Tier-0)

**系统定位**：
为了保护在几十年一遇的“系统性流动性抽干危机”（如 2008 次贷危机、雷曼倒闭）时过度进账，引入 **Tier 0（最高优先级开关）**。它的权限高于一切，能在极端宏观恶化时强制熄灭所有买点。

#### Feature 4: 信用利差危机预警 (Credit Spread Blowout)
*   **监控指标**：
    高收益企业债期权调整利差 (ICE BofA US High Yield Index Option-Adjusted Spread，如 FRED 代码 `BAMLH0A0HYM2`)。
*   **计算逻辑**：
    监控该利差的绝对值与近期（最近 1-3 个月）的飙升斜率。如果信用利差突破预设危险极值（如 500+ 个基点）或出现几乎垂直的爬升。
*   **执行动作**：
    触发 **Tier 0 硬否决**（熔断）。系统无视 Tier 1 全满分和 PW 支撑，强制将信号降级或锁定为 `NO_SIGNAL` / `CRISIS_WARNING`，并在 Discord 输出中强烈警告“宏观底层流动性管道破裂”。此状态需持续至信用利差筑顶回落才可解除。

## 3. 系统架构演进及影响

### 3.1 跨日状态记忆依赖
底背离的运算**强依赖于历史高低点的比对**。这意味着现有的系统需要从“每日无状态即时计算”进化为“时间序列窗口分析”。
*   **改动点**：需要从 SQLite（或引入新的轻量级缓存）中拉取过去 3~6 个月的 QQQ、VIX、Breadth 历史标量数据系列，找出波段的 `(T_min_price, P_min)` 和 `(T_min_breadth, B_min)` 坐标组合。

### 3.2 评分项满分溢出重构
目前的 Tier-1 是满分 100 分。如果加入了最高 +30 分的“背离奖励分（Bonus）”，可能导致总分超过 100。
*   **改动点**：
    1. 放大总分上限（如 120/100）。
    2. 维持原积分权重不变，背离作为“超额确定性”的表现，意味着即使在某些（如 VIX 刚回落）单项未拿满分的情况下，凭借背离红利依然能够满足 70 分以上的 `TRIGGERED` 绿灯判定。

### 3.3 数据源挑战 (FRED API)
宏观信用利差数据无法通过 `yfinance` 稳定获取。
*   **改动点**：需要接入美联储经济数据 (FRED) 的免费 API，并处理宏观数据的长延迟（T+1/T+2）特性，使其能与每日收盘的现货行情完成时间对齐。

## 4. 交付里程碑规划 (v2.0 Roadmap)

- [ ] **M1（数据层升级）**：时间序列化，从 SQLite 回溯 N 日历史数据用以寻找波段高低点和计算 RSI。
- [ ] **M2（背离判定引擎）**：开发独立的 Divergence Engine，计算 Price-Breadth 和 Price-VIX 背离奖励分。引入历史回测对比开启与关闭背离因子的胜率。
- [ ] **M3（宏观熔断器建立）**：集成 FRED API，实现 Tier-0 信用利差熔断器。
- [ ] **M4（全案回测与交付）**：用 2008 年的金融海啸史料，专门验证信用利差是否能在第一时间内锁定买入面板，以及之后何时解禁。升级 Discord 输出样式。
