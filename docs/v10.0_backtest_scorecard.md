## 1. 目的

这份文档定义 v10.0 的正式回测评分卡，用于回答四个问题：

1. 输入数据是否足够可信。
2. 系统输出是否符合 Architect 10.0 的周期决策逻辑。
3. 组合后果是否优于简单基线，并且没有靠高摩擦伪造表现。
4. 候选库是否仍满足离线研究认证约束。

它不是单纯的收益排行榜。v10.0 首先是一个风险预算与部署系统，其次才是一个收益改进系统。

## 2. 评价原则

### 2.1 先看数据，再看收益

如果 macro dataset 不满足 canonical contract，或仍是 synthetic fixture，那么任何收益结论都不成立。

### 2.2 先看决策对齐，再看净值

v10.0 的核心资产不是某条收益曲线，而是：

- `Tier-0 -> Cycle -> Risk -> Candidate -> Advisory -> Deployment`

这条链必须先被历史验证为“按设计运行”，然后才谈组合表现。

### 2.3 左尾和治理优先于进攻

系统的第一职责是：

- 不越过 beta floor / cap
- 不在 `CRISIS` 中无授权地部署
- 不让 `QLD` 在非法周期中恢复常态化使用

### 2.4 候选认证是补充 gate，不替代 backtest

候选库仍要经过 certifier 审查，但 certifier 是 release gate，不是 portfolio backtest score 的替代品。

## 6. Interpretation

### 6.1 这 96 分意味着什么

它说明：

- v10 周期层已经被历史 acceptance 验证为“按设计工作”
- 回测中没有出现 beta envelope 越权
- `CRISIS` 治理约束有效
- tactical path 相对 baseline DCA 有明确左尾改善
- friction 成本可见但仍处于可接受区间

### 6.2 为什么不是 100 分

扣分主要来自数据覆盖，而不是决策失真：

- `real_yield_10y_pct` 覆盖仍不足满分
- `net_liquidity_usd_bn` 与 `liquidity_roc_pct_4w` 仍有明显缺口

这不再阻断 v10 acceptance，因为估值历史已经补齐；但它仍是后续研究质量提升的自然方向。

## 7. Additional Diagnostics

以下指标保留为诊断项，不直接纳入总分：

- `Forward returns` by `T+5 / T+20 / T+60`
- `Max adverse excursion after add`
- `Average cost vs baseline DCA`
- `window-by-window beta_match / deployment_match`

原因：

- 它们对具体时间窗口、市场阶段和信号分布更敏感
- 更适合作为 explainability 与 regression audit 工具，而不是长期主评分项

## 8. Supplementary Gate: Candidate Certification

候选库仍需通过 certifier 的离线研究 gate。相关指标包括：

- `max_drawdown`
- `cagr`
- `ulcer_index`
- `expected_shortfall`
- `mean_interval_beta_deviation`
- `turnover`
- `defense_coverage`
- `nav_integrity`

认证规则：

- `CERTIFIED`: `MDD <= budget`，`nav_integrity >= 0.99`，`beta deviation <= 0.05`
- `CONDITIONAL`: 容忍轻微超限
- `REJECTED`: 其余情形

这部分是 release governance 的必要条件，但不并入当前 backtest 总分。

## 9. Release Rule

v10.0 回测验收建议使用以下 release 规则：

1. Hard gates 全通过
2. Scorecard 总分 `>= 90`
3. Acceptance:
   `beta MAE <= 0.02` 且 `within_tol >= 95%`
4. Governance:
   `unauthorized crisis breaches = 0`
5. Portfolio:
   `Tactical Max Drawdown <= 30%`

满足以上条件，才能称为：

- `spec compliant`
- `architecture aligned`
- `backtest effective`
- `ready for production narrative`
